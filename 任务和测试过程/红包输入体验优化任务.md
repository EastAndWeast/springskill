# 红包输入体验优化任务

## 任务目标
解决用户反馈的：在手机端“记红包”模块下，“谁/给谁”与“金额”两项输入框挤在同一行导致显示不全、操作不便的问题。

## 问题分析
原代码：
```jsx
<div className="flex gap-2">
    <input className="flex-1 ... />
    <input className="w-28 ... />
</div>
```
- 问题1：移动端尤其在类似 iPhone SE 屏幕较窄的设备上，固定宽度 `w-28` 加上输入框 padding 会导致另一个输入框被压得非常小。
- 问题2：左右相邻的输入框导致手机键盘弹出后，视线需要经常横向游移。

## 解决步骤
1. 修改 `RedPacketModule.jsx`。
2. 将 `<div className="flex gap-2">` 改为 `<div className="flex flex-col gap-3">`，使其在所有宽度下都保持纵向排列（这非常符合移动端单手操作的黄金直觉）。
3. 调整输入框的宽度均为 `w-full`（100% 宽），并为“金额”输入框添加人民币符号 `¥` 作为前缀提示，优化 UI 的可用性和美观度。
4. 调整两个输入框在纵向上的边距（gap-3 增加舒适呼吸感）。

## 结果验证
- 手机端视图下（可利用开发者工具模拟），“谁/给谁”占据一行，“金额”占据下一行。
- 操作空间（触控面积）扩大了 100%。
- 输入不会被遮挡或挤压，整体美观和一致性更好。
- 不影响其底层 localStorage 数据持久化的逻辑。
